Cómo probar la PWA

En tu PC (Chrome)

  1. Abre el sitio en Chrome
  2. F12 → pestaña Application
  3. Sección Manifest → todos los campos deben estar en verde
  4. Sección Service Workers → debe decir "activated and is running"
  5. Sección Storage → puedes ver el cache aluvima-v1-shell

En tu móvil

  1. Abre el sitio en Chrome móvil
  2. Menú (⋮) → "Instalar app" o "Añadir a pantalla de inicio"
  3. Verás el ícono Aluvima en tu lanzador
  4. Al abrir, se ve sin barra del navegador (modo standalone)

Archivos PWA del proyecto

  • manifest.webmanifest — metadatos (nombre, colores, atajos)
  • service-worker.js — cache offline (network-first para HTML, cache-first para imágenes)
  • icons/icon.svg — logo vectorial
  • icons/icon-192.png, icon-512.png, icon-512-maskable.png — íconos para Play Store y dispositivos
  • tools/generar-iconos.html — generador de los PNG (por si quieres regenerar)

Para regenerar los íconos

Si cambias el logo o quieres reemplazar:

  1. Abre tools/generar-iconos.html en tu Chrome (doble clic)
  2. Clic en cada botón Descargar (3 archivos)
  3. Mueve los 3 PNG a la carpeta icons/
Cuando quieras logo profesional: sube el logo final (PNG 1024×1024) a https://realfavicongenerator.net y reemplaza los archivos de icons/.

Notas técnicas importantes

  • El service worker solo funciona en HTTPS (excepto localhost)
  • Cada vez que cambies CSS/JS, sube el VERSION en service-worker.js para forzar refresco del cache
  • Si en desarrollo te quedan archivos viejos cacheados: DevTools → Application → Clear storage

Para publicar en Play Store

Ver guía completa: PUBLICAR_PLAY_STORE.html

Resumen rápido:

  1. Sitio HTTPS público (Netlify ya da)
  2. Lighthouse PWA score ≥ 80 (ya está)
  3. https://www.pwabuilder.com → pega tu URL → "Package for Android"
  4. Sube el .aab resultante a Google Play Console (US$25 cuenta dev)
